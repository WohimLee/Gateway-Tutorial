可以按这四类分：

**REST 接口**
```text
GET/HEAD /health
GET/HEAD /healthz
GET/HEAD /ready
GET/HEAD /readyz

POST /tools/invoke
POST /sessions/{sessionKey}/kill

POST {hooks.basePath}/wake
POST {hooks.basePath}/agent
POST {hooks.basePath}/{mappingPath}

/api/channels/mattermost/command
/api/channels/mattermost/*
registered plugin HTTP routes
Slack plugin HTTP routes
```

说明：这些都是普通 HTTP 请求/响应语义。`hooks`、Mattermost、Slack、plugin route 虽然是动态路径或插件入口，但本质仍是 REST/HTTP webhook 风格接口。

**SSE 接口**
```text
GET /sessions/{sessionKey}/history
```

当请求头包含：

```text
Accept: text/event-stream
```

它返回 SSE stream，事件类型：

```text
history
message
```

注意：同一个路径如果不带 `Accept: text/event-stream`，就是普通 REST JSON：

```text
GET /sessions/{sessionKey}/history
```

**OpenAI-compatible 接口**
```text
GET  /v1/models
GET  /v1/models/{modelId}
POST /v1/chat/completions
POST /v1/responses
POST /v1/embeddings
```

说明：这些路径模拟 OpenAI API surface。`/v1/chat/completions` 和 `/v1/responses` 需要配置启用；`/v1/embeddings` 也是 OpenAI 风格接口。

**静态资源接口**
```text
GET/HEAD /__openclaw__/a2ui...
GET/HEAD /__openclaw__/canvas...

{controlUiBasePath}...
```

说明：这些主要返回 HTML、JS、CSS、图片等静态资源。`controlUiBasePath` 是 Control UI SPA 的入口和静态资源路径。


对，**SSE 属于 HTTP**。

更准确地说：

```text
SSE = Server-Sent Events
```

它是基于普通 HTTP 的一种单向流式响应协议。

客户端还是发一个普通 HTTP 请求：

```http
GET /sessions/{sessionKey}/history
Accept: text/event-stream
```

服务端不立刻结束响应，而是持续写入：

```text
event: history
data: {...}

event: message
data: {...}
```

所以分类上可以这样理解：

```text
HTTP
├── REST/普通 JSON 接口
├── SSE 流式接口
├── OpenAI-compatible HTTP 接口
└── 静态资源接口
```

而 WebSocket 不一样，它一开始也是 HTTP Upgrade，但升级成功后就不再是普通 HTTP 请求/响应模型了。
