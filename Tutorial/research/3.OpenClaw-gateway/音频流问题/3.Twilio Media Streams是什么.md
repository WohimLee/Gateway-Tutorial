Twilio Media Streams 是 Twilio Voice 的一个实时音频接口：**电话接通后，Twilio 把通话里的原始音频通过 WebSocket 发给你的服务；如果是双向流，你的服务也可以把音频通过同一个 WebSocket 发回 Twilio，让对方在电话里听到。**

可以把它理解成：

```text
电话用户
  <-> Twilio 电话网络
  <-> WebSocket
  <-> 你的服务 / STT / AI agent / TTS
```

它不是 WebRTC，也不是普通录音文件下载，而是实时 WebSocket 流。

典型消息是 JSON：

```json
{
  "event": "media",
  "streamSid": "MZ...",
  "media": {
    "payload": "base64..."
  }
}
```

`payload` 里面是 base64 编码的原始音频。Twilio 文档里写的格式是：

- encoding: `audio/x-mulaw`
- sample rate: `8000`
- channels: `1`

也就是电话常见的 **G.711 μ-law，8kHz，单声道**。

它有两种模式：

- **单向 Media Stream**：Twilio 把电话音频发给你。适合实时转写、情绪分析、质检、语音认证。
- **双向 Media Stream**：Twilio 把电话音频发给你，你也可以把音频发回 Twilio。适合 AI 电话助手、实时 IVR、语音机器人。

在 OpenClaw 这里，`voice-call` 插件用的就是这类模型：

```text
Twilio Media Streams
  -> /voice/stream WebSocket
  -> OpenAI Realtime STT
  -> agent
  -> TTS
  -> WebSocket 发回 Twilio
```

所以它的价值是：**你不用自己接运营商电话网络，也不用自己做 WebRTC 电话 SDK；Twilio 负责电话侧，你只处理 WebSocket 音频帧。**
