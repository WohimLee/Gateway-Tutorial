查了源码：**OpenClaw 现在没有 WebRTC 接口**。仓库里没有 `RTCPeerConnection` / WebRTC 信令 / SDP / ICE 这类实现；gateway 的主控制面是 **WebSocket JSON RPC**，文档也写的是 Gateway WebSocket protocol，见 `docs/gateway/protocol.md`。

音频相关分几类：

**1. Gateway 主接口不是音频流，是 WS 控制/RPC**

核心 gateway 走：

- `ws://...` / `wss://...`
- JSON text frames
- `connect.challenge` → `connect` → `req/res/event`

对应文档在 `docs/gateway/protocol.md`，runbook 里也写主端口承载 WebSocket control/RPC 和 HTTP API：`docs/gateway/index.md`。

**2. 电话语音流走 WebSocket，不走 WebRTC**

`voice-call` 插件里有实时媒体流接口，默认路径是：

- `/voice/stream`
- WebSocket upgrade
- Twilio Media Streams 风格消息

代码在：

- `extensions/voice-call/src/config.ts`
- `extensions/voice-call/src/webhook.ts`
- `extensions/voice-call/src/media-stream.ts`

音频格式是 **G.711 μ-law / 8kHz mono**。入站流程大概是：

```text
Twilio Media Stream
  -> WebSocket /voice/stream
  -> base64 μ-law audio
  -> OpenAI Realtime STT WebSocket
```

关键代码在 `extensions/voice-call/src/media-stream.ts`：它接收 Twilio 的 `media.payload`，base64 解码后 `sttSession.sendAudio(audioBuffer)`。

STT 侧在 `extensions/voice-call/src/providers/stt-openai-realtime.ts`，连接：

```text
wss://api.openai.com/v1/realtime?intent=transcription
```

并配置：

```text
input_audio_format: "g711_ulaw"
```

所以这里也不是 WebRTC，是 **WebSocket 到 OpenAI Realtime STT**。

**3. TTS 回电话也是 μ-law 8kHz 通过同一个 WebSocket 发回去**

TTS 生成后会转成 μ-law 8kHz，再发 Twilio media frame：

- 转码：`extensions/voice-call/src/telephony-audio.ts`
- TTS 包装：`extensions/voice-call/src/telephony-tts.ts`
- 发送：`extensions/voice-call/src/media-stream.ts`

发送形状是：

```json
{
  "event": "media",
  "streamSid": "...",
  "media": { "payload": "base64..." }
}
```

**4. App 的 Talk/Voice 不是把麦克风 PCM 流推给 gateway**

iOS/Android 的 talk mode 看起来主要是本机 speech recognition + gateway `chat.send` + 本机/服务端 TTS 播放。比如 iOS 里用 `SFSpeechRecognizer`、`AVAudioEngine`，再通过 gateway 发送文本；TTS 可以直接流式拉 ElevenLabs 音频播放，见 `apps/ios/Sources/Voice/TalkModeManager.swift`。

所以一句话总结：

**没有 WebRTC。OpenClaw 的实时控制面走 Gateway WebSocket；电话音频流也是 WebSocket，Twilio Media Streams 的 base64 μ-law 8kHz，STT 再转发到 OpenAI Realtime WebSocket。**
