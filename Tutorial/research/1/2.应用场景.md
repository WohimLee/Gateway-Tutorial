# 五、从“场景角度”的选择逻辑（非常实用）

可以用这个决策思路👇

---

### ① 只请求一次？

👉 用 **HTTP**

---

### ② 需要“边返回边显示”？（AI、下载）

👉 用 **Streaming HTTP**

---

### ③ 只需要服务器推送？

👉 用 **SSE**

---

### ④ 双方都要实时互动？

👉 用 **WebSocket**

---

# 六、真实项目中的组合（重点）

很多系统不是只用一种，而是👇

### 常见组合

* HTTP → 登录 / API
* SSE → 实时通知 / AI输出
* WebSocket → 聊天 /协作

👉 比如 ChatGPT：

* 请求：HTTP
* 输出：Streaming / SSE
